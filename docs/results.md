# Results

PolicyBench results are generated by the production analysis pipeline, not by embedded notebook cells or ad hoc exploratory code.

Run the benchmark and analysis workflow:

```bash
# Defaults to the rebuilt v2_headline output scope.
policybench ground-truth -n 100 --seed 42
policybench eval-no-tools -n 100 --seed 42
policybench analyze --output-dir results/local/analysis
```

The `analyze` command writes four local artifacts:

1. `results/local/analysis/metrics.csv`
2. `results/local/analysis/summary_by_model.csv`
3. `results/local/analysis/summary_by_variable.csv`
4. `results/local/analysis/report.md`

## What to read

- `summary_by_model.csv` is the equal-output summary for the no-tools benchmark.
- `impact_summary_by_model.csv` is the household-equal impact-weighted summary when available.
- `summary_by_variable.csv` shows which policy variables are easiest and hardest.
- `report.md` is a compact Markdown summary suitable for publishing or review.

## Interpretation

Three patterns matter most when reading these outputs:

1. The benchmark is intentionally about unaided capability. Strong results here mean a model can estimate selected household-level policy outputs from parametric knowledge alone.
2. The rebuilt headline scope focuses on signed household net-income components; intermediate tax bases are supplementary diagnostics.
3. Amount variables are scored with exact and tolerance hit rates. Coverage variables are scored as booleans and weighted by PolicyEngine value proxies in the household-equal impact score.

## Reproducibility

Reference outputs and predictions should always be generated from the same sampled scenarios by using the same `--seed` and `-n/--num-scenarios` values. The exported analysis artifacts are deterministic functions of those two input files:

- `results/local/ground_truth.csv`
- `results/local/no_tools/predictions.csv`
